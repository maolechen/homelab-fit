# SEO Positioning

## Avoid

Do not compete head-on with official documentation for broad queries:

- `proxmox install`
- `proxmox storage`
- `proxmox networking`
- `proxmox zfs`

## Prioritize

Target concrete buying and troubleshooting searches:

- `intel n100 proxmox nas`
- `minisforum ms-01 proxmox`
- `proxmox no network after install`
- `proxmox vm no internet`
- `proxmox zfs mirror mini pc`
- `proxmox hba passthrough truenas`
- `n100 vs n305 proxmox`
- `proxmox mini pc vs used server`
- `proxmox realtek rtl8125`
- `proxmox vlan aware bridge`

## Page Positioning Rule

Every indexable page should answer one of these:

- Should I buy this hardware for Proxmox?
- Will this NIC/storage/passthrough setup fit my build?
- What should I check before I change this configuration?
- What failure mode explains this exact symptom?
- Which architecture is simpler or safer for this workload?

## Differentiation

The value is structured decision support:

- Fit criteria
- Not-fit warnings
- Verification checks
- Recovery planning
- Related build paths
- Source links and access dates
